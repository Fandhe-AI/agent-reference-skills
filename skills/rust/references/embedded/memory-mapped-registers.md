# Memory-Mapped Registers

Peripherals on microcontrollers are accessed by reading/writing specific memory addresses. The Rust embedded ecosystem provides multiple abstraction layers on top of raw pointer operations.

## Abstraction Layers (bottom to top)

| Layer | Crate Example | Purpose |
|-------|--------------|---------|
| Raw pointer | — | Direct `unsafe` memory access |
| Micro-architecture | `cortex-m` | Processor-core peripherals (SysTick, NVIC, etc.) |
| Peripheral Access Crate (PAC) | `stm32f30x`, `tm4c123x` | SVD-generated register wrappers |
| HAL crate | `stm32f3xx-hal` | High-level, user-friendly API |
| Board crate | `stm32f3-discovery` | Pre-configured pins for a specific board |

## Raw Struct Approach (avoid in production)

```rust
#[repr(C)]
struct SysTick {
    pub csr: u32,
    pub rvr: u32,
    pub cvr: u32,
    pub calib: u32,
}

// Every access requires unsafe and volatile reads
let systick = unsafe { &mut *(0xE000_E010 as *mut SysTick) };
let time = unsafe { core::ptr::read_volatile(&systick.cvr) };
```

Use `volatile_register::{RW, RO}` to encode read/write permissions in types.

## PAC: read / write / modify

```rust
use tm4c123x;

let p = tm4c123x::Peripherals::take().unwrap();
let pwm = p.PWM0;

// read() — returns typed bitfield accessor
if pwm.ctl.read().globalsync0().is_set() { /* ... */ }

// write() — closure sets fields; unspecified fields reset to default
pwm.ctl.write(|w| w.globalsync0().clear_bit());

// modify() — closure gets (read, write); unspecified fields preserved
pwm.ctl.modify(|_r, w| w.globalsync0().clear_bit());
```

PAC code is auto-generated by `svd2rust` from vendor SVD files.

## Micro-architecture Crate (cortex-m)

```rust
#![no_std]
#![no_main]

use cortex_m::peripheral::{syst, Peripherals};
use cortex_m_rt::entry;
use panic_halt as _;

#[entry]
fn main() -> ! {
    let peripherals = Peripherals::take().unwrap();
    let mut systick = peripherals.SYST;
    systick.set_clock_source(syst::SystClkSource::Core);
    systick.set_reload(1_000);
    systick.clear_current();
    systick.enable_counter();
    while !systick.has_wrapped() {}
    loop {}
}
```

## HAL Crate

```rust
use tm4c123x_hal as hal;
use tm4c123x_hal::prelude::*;
use tm4c123x_hal::serial::{NewlineMode, Serial};

#[entry]
fn main() -> ! {
    let p = hal::Peripherals::take().unwrap();
    let mut sc = p.SYSCTL.constrain();
    // ... clock configuration ...
    let clocks = sc.clock_setup.freeze();
    let mut porta = p.GPIO_PORTA.split(&sc.power_control);

    let uart = Serial::uart0(
        p.UART0,
        porta.pa1.into_af_push_pull::<hal::gpio::AF1>(&mut porta.control),
        porta.pa0.into_af_push_pull::<hal::gpio::AF1>(&mut porta.control),
        (), (), 115200_u32.bps(),
        NewlineMode::SwapLFtoCRLF,
        &clocks, &sc.power_control,
    );
    loop { writeln!(uart, "Hello\r\n").unwrap(); }
}
```

## Notes

- Always use `modify()` when changing only some bits; `write()` resets unspecified fields to defaults
- PAC types prevent register access before peripheral is enabled (compile-time safety)
- `unsafe` is required for raw pointer access but not for PAC/HAL APIs
- Board crates are most convenient for beginners but tie code to specific hardware
