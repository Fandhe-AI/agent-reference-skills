# Instrument Coverage

Instrumentation-based code coverage uses LLVM profiling to measure which parts of Rust code are executed. Enabled via `-C instrument-coverage`.

## Signature / Usage

```bash
# Build with coverage instrumentation
RUSTFLAGS="-C instrument-coverage" cargo build

# Run instrumented binary (generates .profraw file)
./target/debug/myprogram

# Merge profiles
llvm-profdata merge -sparse default.profraw -o coverage.profdata

# Generate summary report
llvm-cov report --instr-profile=coverage.profdata ./target/debug/myprogram
```

## Options / Props

### `-C instrument-coverage[=<VALUE>]`

| Value | Description |
|-------|-------------|
| (no value) / `yes` / `y` / `on` / `true` | Enable coverage for all functions |
| `all` | Currently identical to `yes`; may differ in future |
| `no` / `n` / `off` / `false` | Disable coverage (default) |

### `LLVM_PROFILE_FILE` Environment Variable

Controls the output filename of the `.profraw` file:

| Pattern | Substitution |
|---------|-------------|
| `%p` | Process ID |
| `%h` | Hostname |
| `%t` | Value of `$TMPDIR` |
| `%Nm` | Binary signature (pool size N, 1â€“9; default 1) |
| `%c` | Enable continuous mode (sync on each write, prevents data loss on crash) |

```bash
LLVM_PROFILE_FILE="coverage_%m_%p.profraw" ./myprogram
```

### `-Z coverage-options=<OPTIONS>`

Unstable fine-grained coverage control (nightly only). See the Unstable Book for details.

## Prerequisites

```bash
# Install LLVM tools
rustup component add llvm-tools-preview

# Install rustfilt demangler (optional, for readable stack traces)
cargo install rustfilt
```

## Generating Reports

### Summary Report

```bash
llvm-cov report \
  --instr-profile=coverage.profdata \
  ./target/debug/myprogram
```

### Detailed Source Report

```bash
llvm-cov show \
  -Xdemangler=rustfilt \
  ./target/debug/myprogram \
  -instr-profile=coverage.profdata \
  -show-line-counts-or-regions \
  -show-instantiations
```

### Test Coverage

```bash
RUSTFLAGS="-C instrument-coverage" cargo test --tests

# Merge all profraw files generated by test binaries
llvm-profdata merge -sparse default_*.profraw -o tests.profdata

llvm-cov report \
  --use-color --ignore-filename-regex='/.cargo/registry' \
  --instr-profile=tests.profdata \
  --object target/debug/deps/mylib-<HASH>
```

### Including Doc Tests

```bash
RUSTFLAGS="-C instrument-coverage" \
RUSTDOCFLAGS="-C instrument-coverage -Z unstable-options --persist-doctests target/debug/doctestbins" \
  cargo test

llvm-profdata merge -sparse default_*.profraw -o all.profdata

llvm-cov report \
  --instr-profile=all.profdata \
  --object target/debug/deps/mylib-* \
  --object target/debug/doctestbins/*/rust_out
```

## Coverage Metrics

| Metric | Description |
|--------|-------------|
| Function coverage | % of functions executed at least once |
| Instantiation coverage | % of generic/macro instantiations executed |
| Line coverage | % of executable lines executed |
| Region coverage | % of code regions executed (most granular) |

## Disabling Coverage on Specific Functions

```rust
#![feature(coverage)]  // nightly only

#[coverage(off)]
fn skip_coverage() {
    // excluded from coverage reports
}
```

## Notes

- `-C instrument-coverage` automatically enables `-C symbol-mangling-version=v0`.
- Requires LLVM 12+; use the `llvm-profdata` version that matches the compiler's LLVM.
- Use `%m` and `%p` patterns in `LLVM_PROFILE_FILE` when running multiple test binaries to avoid overwriting.
- Prior to Rust 1.65, the default output file was `default.profraw`.
