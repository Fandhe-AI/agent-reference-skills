# Rust 2018

**Theme: Productivity** — Released with Rust 1.31.0 (December 6, 2018). RFC [#2052](https://rust-lang.github.io/rfcs/2052-epochs.html).

## Signature / Usage

```toml
[package]
edition = "2018"
```

Migrate with:

```bash
cargo fix --edition
```

---

## Breaking Changes

### 1. Path and Module System

#### No more `extern crate`

```rust
// 2015 — required
extern crate futures;
use futures::Future;

// 2018 — just add to Cargo.toml, no extern crate needed
use futures::Future;
```

Exceptions that still need `extern crate`: `std` (implicit unless `#![no_std]`), `alloc` (for `no_std` + allocation), `test` (nightly), `proc_macro` (Cargo < 1.42).

Macro imports also switch to `use`:

```rust
// 2015
#[macro_use] extern crate bar;
baz!();

// 2018
use bar::baz;
baz!();
```

#### Crate renaming without `extern crate`

```rust
// 2018
use futures as f;
use self::f::Future;
```

#### `crate::` keyword

Refer to the current crate root unambiguously from anywhere:

```rust
crate::foo::bar  // always the crate root, in any module
```

`::foo` now unambiguously refers to an external crate named `foo`.

#### No more `mod.rs`

```
# 2015 structure
src/
  lib.rs
  foo/
    mod.rs
    bar.rs

# 2018 — mod.rs optional, foo.rs can coexist with foo/
src/
  lib.rs
  foo.rs
  foo/
    bar.rs
```

#### Uniform `use` paths

`use` paths and regular paths now follow the same rules everywhere. A path that works at the crate root also works inside a submodule without modification.

---

### 2. Anonymous Trait Function Parameters Deprecated

```rust
// 2015 — allowed
trait Foo { fn foo(&self, u8); }

// 2018 — must name (or use _)
trait Foo { fn foo(&self, _: u8); }
```

Automated by `cargo fix --edition`.

---

### 3. New Keywords

| Keyword | Status in 2018 | Notes |
|---------|----------------|-------|
| `dyn` | Strict keyword | Required for trait objects |
| `async` | Strict keyword | Stabilized in Rust 1.39 |
| `await` | Strict keyword | Stabilized in Rust 1.39 |
| `try` | Reserved | Not yet stabilized |

#### `dyn Trait` replaces bare trait objects

```rust
// 2015 (still a warning)
fn f() -> Box<Trait> { unimplemented!() }

// 2018 (required)
fn f() -> Box<dyn Trait> { unimplemented!() }
```

`dyn` makes it explicit that dynamic dispatch is used, and creates visual symmetry with `impl Trait`.

#### Identifiers conflicting with new keywords

Code using `async`, `await`, or `try` as identifiers must be rewritten. `cargo fix` converts them to raw identifiers (`r#async`, etc.).

---

### 4. Method Dispatch for Raw Pointers to Inference Variables

Method calls on raw pointers to inference variables now require a concrete type to be known before dispatch. In rare cases this changes which method is selected. Automated by `cargo fix --edition`.

---

### 5. Cargo Changes

- `Cargo.lock` is now recommended to be committed for binary packages.
- The `edition` field in `Cargo.toml` becomes the primary mechanism for opting in to new behaviours.

---

## Notes

- Most Rust 2018 features are also available on all editions (they are not edition-gated at the language level); only the module system changes and keyword strictness are edition-specific.
- `cargo fix --edition` handles the vast majority of 2018 migration automatically.
- Minimum Rust version for 2018 edition: **1.31.0**.
