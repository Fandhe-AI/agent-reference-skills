# Rust 2021

**Theme: Consistency & capability** — Released with Rust 1.56.0 (October 21, 2021). RFC [#3085](https://github.com/rust-lang/rfcs/pull/3085).

## Signature / Usage

```toml
[package]
edition = "2021"
```

Migrate with:

```bash
cargo fix --edition
```

---

## Breaking Changes

### 1. Additions to the Prelude

Three traits are added to `std::prelude`:

- `std::convert::TryInto`
- `std::convert::TryFrom`
- `std::iter::FromIterator`

**Impact**: If your code has in-scope traits with methods named `try_into`, `try_from`, or `from_iter`, calling those methods may become ambiguous and fail to compile.

```rust
// Breaks if MyTrait also defines try_into()
let v: MyType = x.try_into()?;
```

**Fix — use fully qualified syntax**:

```rust
<MyType as MyTrait>::try_into(x)
```

**Fix for `dyn Trait` objects** (the trait cannot be imported implicitly for dyn dispatch):

```rust
// Before
fn bar(f: Box<dyn MyTrait>) { f.try_into(); }

// After — add explicit deref to prefer the inherent/dyn method
fn bar(f: Box<dyn MyTrait>) { (&*f).try_into(); }
```

Inherent methods always take priority over trait methods and require no change.

Automated by the `rust_2021_prelude_collisions` lint via `cargo fix --edition`.

### 2. Default Cargo Feature Resolver (`resolver = "2"`)

`edition = "2021"` implies `resolver = "2"` automatically. The new resolver does **not** merge features across different dependency kinds:

| Scenario | 2018 behaviour | 2021 behaviour |
|----------|----------------|----------------|
| Build-dep and normal dep share features | Yes | No — built separately |
| Dev-dep features activate for normal builds | Yes | No |
| Platform-specific dep features activated on all targets | Yes | No |

**Fix**: Explicitly list required features on each dependency:

```toml
[dependencies]
bstr = { version = "0.2", default-features = false, features = ["unicode"] }
```

No automated fix available — use `cargo tree -e features` to audit.

### 3. `IntoIterator` for Arrays

Since Rust 1.53, arrays implement `IntoIterator`. In 2021 edition the change takes full effect:

```rust
let array = [1u8, 2, 3];
for x in array.into_iter() {
    // 2015/2018: x is &u8  (resolved to (&array).into_iter())
    // 2021:      x is u8   (true IntoIterator on the array)
}
```

**Fix** — to preserve reference semantics:

```rust
for x in array.iter() { /* x is &u8 in all editions */ }
```

Automated by the `array_into_iter` lint via `cargo fix --edition`.

### 4. Disjoint Capture in Closures

Closures now capture **individual fields** they use, not the whole variable.

```rust
let a = SomeStruct::new();

// 2018: closure captures all of `a`, so moving a.x first is an error
// 2021: closure captures only `a.y`, so moving a.x first is OK
drop(a.x);
let c = || println!("{}", a.y); // compiles in 2021
```

**Drop order**: Captured fields drop when the closure drops, not when the original variable's scope ends. This can change observable drop order.

**Trait impls**: A closure's `Send`/`Sync`/`Clone` bounds now depend only on the captured fields.

**Wild-card patterns**: `let _ = x;` inside a closure does **not** capture `x`.

`cargo fix --edition` inserts `let _ = &x;` ("dummy let") inside the closure body when it detects a drop-order or trait-impl change.

```rust
// cargo fix may produce:
let c = move || {
    let _ = &x; // forces whole-variable capture
    println!("{:?}", x.0);
};
```

Review inserted dummy lets — many can be safely removed after manual inspection.

### 5. `panic!` Macro Consistency

`panic!` now always behaves like `println!` (uses `format_args!`):

| Expression | 2018 | 2021 |
|------------|------|------|
| `panic!("{}", 1)` | OK | OK |
| `panic!("{}")` | OK (literal `{}`) | Error: missing argument |
| `panic!(x)` | OK if non-string | Error: must be string literal |
| `panic!(123)` | OK | Error |

**Fixes**:

```rust
// For non-string payloads:
std::panic::panic_any(MyStruct);

// For values:
panic!("{}", x);

// For literal braces in message:
panic!("Some curlies: {{}}");
```

Automated by the `non_fmt_panics` lint via `cargo fix --edition`.

### 6. Reserved Syntax (Prefixed Identifiers)

The following patterns are now **reserved** for future language expansion and no longer tokenize:

```
ident#, ident"...", ident'...', 'ident#
```

Impact is mainly on macros that relied on adjacency of these tokens:

```rust
// 2018 — tokenizes as two tokens
quote!{ #a#b }
my_macro!(z"hey")

// 2021 — add whitespace
quote!{ #a #b }
my_macro!(z "hey")
```

Automated by the `rust_2021_prefixes_incompatible_syntax` lint.

---

### 7. Raw Lifetimes

`'r#ident` is now a single token (a raw lifetime). In 2015/2018 it was three tokens (`'r`, `#`, `ident`). Macro code relying on that tokenization needs whitespace.

Automated by `cargo fix --edition`.

---

### 8. Warnings Promoted to Errors

| Lint | Effect |
|------|--------|
| `bare_trait_objects` | `&Trait` / `Box<Trait>` must be written `&dyn Trait` / `Box<dyn Trait>` |
| `ellipsis_inclusive_range_patterns` | `...` in patterns must be `..=` |

```rust
// 2021 errors
fn f(_: &MyTrait) {}           // must be &dyn MyTrait
matches!(n, 0...100)           // must be 0..=100
```

Automated by `cargo fix --edition`.

---

### 9. Or Patterns in `macro_rules!`

`:pat` fragment specifier now matches `A | B` (top-level `|`). Use `:pat_param` to retain old behaviour when the pattern is followed by an explicit `|`:

```rust
// 2018
macro_rules! m { ($x:pat | $y:pat) => {} }

// 2021
macro_rules! m { ($x:pat_param | $y:pat) => {} }
```

Automated by `cargo fix --edition`.

---

### 10. C-String Literals (Rust 1.77+)

`c"foo"` and `cr"foo"` are now first-class token types producing `&core::ffi::CStr`. Previously `c"..."` tokenized as identifier `c` + string literal. Code with macros relying on that adjacency must add whitespace.

## Notes

- `cargo fix --edition` handles the majority of changes automatically. The notable exception is the Cargo feature resolver — that requires manual dependency review.
- Minimum Rust version for 2021 edition: **1.56.0**.
