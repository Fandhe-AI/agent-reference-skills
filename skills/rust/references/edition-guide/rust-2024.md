# Rust 2024

**Released with Rust 1.85.0 (February 20, 2025). RFC [#3501](https://rust-lang.github.io/rfcs/3501-edition-2024.html).**

## Signature / Usage

```toml
[package]
edition = "2024"
```

Migrate with:

```bash
cargo fix --edition
```

---

## Breaking Changes â€” Language

### 1. RPIT Lifetime Capture Rules

Return-position `impl Trait` (RPIT) now **captures all in-scope generic parameters by default**, including lifetime parameters.

```rust
// In 2021:  lifetimes captured only if they appear syntactically in a bound
// In 2024:  ALL in-scope lifetimes captured unconditionally

fn f<'a>(x: &'a ()) -> impl Sized { *x }
// 2021 equivalent: impl Sized + use<>
// 2024 equivalent: impl Sized + use<'a>
```

Use the explicit `use<..>` bound (stable since 1.82) to control capturing precisely:

```rust
// Opt out of capturing 'a in 2024
fn f<'a>(x: &'a ()) -> impl Sized + use<> { *x }
```

**Migrating away from the `Captures` trick**:

```rust
// Old pattern (all editions)
pub trait Captures<T: ?Sized> {}
impl<T: ?Sized, U: ?Sized> Captures<T> for U {}
fn f<'a, T>(x: &'a (), y: T) -> impl Sized + Captures<(&'a (), T)> { (x, y) }

// Modern (all editions since 1.82)
fn f<'a, T>(x: &'a (), y: T) -> impl Sized + use<'a, T> { (x, y) }

// Rust 2024 (simplified â€” captures everything by default)
fn f<'a, T>(x: &'a (), y: T) -> impl Sized { (x, y) }
```

**APIT (argument-position `impl Trait`) requires manual naming**:

```rust
// Before (overcaptures in 2024)
fn f<'a>(x: &'a (), y: impl Sized) -> impl Sized { (*x, y) }

// After (name the type parameter explicitly)
fn f<'a, T: Sized>(x: &'a (), y: T) -> impl Sized + use<T> { (*x, y) }
```

Detected by the `impl_trait_overcaptures` lint. Automated by `cargo fix --edition` except for APIT cases.

### 2. Tail Expression Temporary Scope

Temporaries in **tail expressions** (the last expression in a function, closure, or block) are now **dropped before local variables** instead of after.

```rust
use std::cell::RefCell;

fn f() -> usize {
    let c = RefCell::new("..");
    c.borrow().len()
    // 2021: Ref temporary dropped AFTER `c` â†’ "does not live long enough" error
    // 2024: Ref temporary dropped BEFORE `c` â†’ compiles fine
}
```

Temporaries may also no longer extend outside a block expression:

```rust
// Fails in 2024, works in 2021
let x = { &String::from("1234") }.len();

// Fix: extract to a local variable
let s = String::from("1234");
let x = s.len();
```

Detected by the `tail_expr_drop_order` lint. **No automated fix** â€” manual review required.

### 3. `unsafe` on `extern` Blocks

`extern` blocks must now be marked `unsafe`:

```rust
// 2021
extern "C" {
    fn strlen(s: *const i8) -> usize;
}

// 2024
unsafe extern "C" {
    // Items can now be individually marked safe or unsafe
    pub safe fn sqrt(x: f64) -> f64;
    pub unsafe fn strlen(p: *const std::ffi::c_char) -> usize;
    pub fn free(p: *mut core::ffi::c_void); // defaults to unsafe
}
```

`safe` items in `unsafe extern` blocks can be called without an `unsafe` block.

Detected by `missing_unsafe_on_extern`. Automated by `cargo fix --edition` (but cannot verify signature correctness â€” manual review required).

---

### 4. `unsafe_op_in_unsafe_fn` Warning Promoted

In 2024 `unsafe_op_in_unsafe_fn` is **warn by default**. Unsafe operations inside `unsafe fn` must be wrapped in an explicit `unsafe` block:

```rust
// 2021 â€” allowed
unsafe fn get<T>(x: &[T], i: usize) -> &T {
    x.get_unchecked(i)
}

// 2024 â€” must use explicit unsafe block
unsafe fn get<T>(x: &[T], i: usize) -> &T {
    unsafe { x.get_unchecked(i) }
}
```

RFC [#2585](https://rust-lang.github.io/rfcs/2585-unsafe-block-in-unsafe-fn.html). Automated by `cargo fix --edition`.

---

### 5. Unsafe Attributes

The attributes `export_name`, `link_section`, and `no_mangle` must now be marked `unsafe`:

```rust
// 2021
#[no_mangle]
pub fn example() {}

// 2024
// SAFETY: no other global function named `example`
#[unsafe(no_mangle)]
pub fn example() {}
```

Detected by `unsafe_attr_outside_unsafe`. Automated by `cargo fix --edition`.

---

### 6. `gen` Keyword Reserved

`gen` is now a reserved keyword (RFC [#3513](https://rust-lang.github.io/rfcs/3513-gen-blocks.html)) in preparation for generator/coroutine blocks in a future release.

```rust
// 2021 â€” allowed as identifier
fn gen() { println!("generating!"); }

// 2024 â€” must use raw identifier
fn r#gen() { println!("generating!"); }
r#gen();
```

Detected by `keyword_idents_2024`. Automated by `cargo fix --edition`.

---

### 7. Never Type Fallback Changes

When the compiler needs a fallback for an unconstrained `!` (never type) at a coercion site, it now falls back to `!` instead of `()`.

```rust
// Pattern that breaks in 2024:
fn f<T: Default>() -> Result<T, ()> { Ok(T::default()) }
f()?;  // T was inferred as () in 2021; now inferred as !, which may not impl Default
```

**Fixes**:

```rust
// Explicit type annotation
f::<()>()?;

// Or use turbofish / type ascription
() = f()?;

// For closures returning !
run(|| -> () { panic!() });
```

The `never_type_fallback_flowing_into_unsafe` lint is `deny` by default in 2024. **No automated fix** â€” manual type annotation required.

## Breaking Changes â€” Cargo

### Rejecting Future-Incompatible Packages

With `edition = "2024"`, Cargo will error (rather than warn) when a dependency is known to have future-incompatibility issues with the current toolchain. Previously this was advisory only.

---

## Breaking Changes â€” Rustfmt

### Style Edition

Rustfmt introduces a `style_edition` that tracks Rust editions. With `edition = "2024"`, `cargo fmt` uses the 2024 style by default:

```bash
cargo fmt           # uses 2024 style when edition = "2024" in Cargo.toml
rustfmt --edition 2024 src/lib.rs
```

Override via `rustfmt.toml`:

```toml
style_edition = "2024"
```
