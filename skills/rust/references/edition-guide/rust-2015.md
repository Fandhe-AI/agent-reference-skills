# Rust 2015

**Theme: Stability** — The original edition, released with Rust 1.0 (May 2015). It is the implicit default for any crate that does not specify `edition` in `Cargo.toml`.

## Key Characteristics

- Establishes Rust's core stability commitment: code that compiles on Rust 1.0 must continue to compile on later releases.
- All subsequent editions remain fully interoperable with 2015 crates.
- No migration to Rust 2015 is ever needed; it is the starting point.

## Module System (2015 behaviour)

- `extern crate foo;` is required to bring a dependency into scope.
- `use` paths are relative to the crate root, not the current module, causing subtle differences between `use` and other path expressions in submodules.
- `mod.rs` files are required to define a module that has sub-modules.
- Trait objects can be written as bare `Trait` (without `dyn`).

## Notes

- Rust 2015 crates can depend on and be depended on by crates compiled with any other edition.
- Code that predates the edition system is automatically treated as Rust 2015.
- Users will generally migrate away from 2015 to 2018, 2021, or 2024 to benefit from cleaner syntax and new features.
