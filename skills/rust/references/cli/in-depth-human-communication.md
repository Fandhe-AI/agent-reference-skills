# In-Depth: Human Communication

Guidelines and crates for producing consistent, informative output that humans can read and follow — covering log levels, progress steps, panic messages, and man page generation.

## Signature / Usage

```rust
// Friendly panic messages with human-panic
use human_panic::setup_panic;
fn main() {
    setup_panic!();
    // ... rest of main
}

// Structured step output (example pattern from wasm-pack)
// [1/7] Adding WASM target...
// [2/7] Compiling to WASM...
//  > [WARN]: Field `description` is missing from Cargo.toml.

// Man page generation in build.rs (clap + clap_mangen)
use clap::CommandFactory;
fn main() -> std::io::Result<()> {
    let out_dir = std::path::PathBuf::from(std::env::var_os("OUT_DIR").unwrap());
    let cmd = cli::Head::command();
    let man = clap_mangen::Man::new(cmd);
    let mut buffer: Vec<u8> = Default::default();
    man.render(&mut buffer)?;
    std::fs::write(out_dir.join("head.1"), buffer)?;
    Ok(())
}
```

## Options / Props

| Crate | Purpose | Cargo.toml |
|-------|---------|------------|
| `human-panic` | User-friendly panic report with crash file | `human-panic = "1"` |
| `clap_mangen` | Generate `man` pages from `clap` `Command` | `clap_mangen = "0.2"` |
| `clap_complete` | Generate shell completion scripts | `clap_complete = "4"` |
| `log` + `env_logger` | Structured log output with `RUST_LOG` | see [human-communication.md](./human-communication.md) |

## Notes

- **Consistency is key**: use the same prefix format, verb tense, and sentence structure across all log and status messages so users can skim them.
- **Log levels**: treat `info` as the default visible level; use `debug`/`trace` for developer details gated behind `--verbose` or `RUST_LOG`.
- **Progress steps**: numbered `[N/M]` steps (as in wasm-pack) give users a clear sense of how far along the process is.
- **Panics**: the default Rust panic message (`thread 'main' panicked at ...`) is useful for developers but confusing for end users; `human-panic` replaces it with a friendly message and writes a report file.
- **Man pages**: generated via `clap_mangen` in `build.rs`; the resulting `.1` file can be included in distribution archives.
- Shell completions generated by `clap_complete` can be included in the same archive so users can install them alongside the binary.
