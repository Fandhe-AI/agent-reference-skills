# In-Depth: Rendering Documentation

CLI applications typically ship two forms of documentation: the built-in `--help` output and system man pages. Both can be generated automatically from `clap` argument definitions using `clap_mangen` in a `build.rs` script.

## Dependencies

```toml
[dependencies]
clap = { version = "4", features = ["derive"] }

[build-dependencies]
clap_mangen = "0.2"
```

## Usage

### Define CLI with `clap` derive

```rust
use clap::Parser;

/// A simple CLI tool
#[derive(Parser)]
#[command(name = "head", about = "Print the first lines of a file")]
struct Cli {
    /// Number of lines to print
    #[arg(short, long, default_value_t = 10)]
    lines: usize,

    /// Input file
    file: String,
}
```

### Generate man page in `build.rs`

```rust
// build.rs
use clap::CommandFactory;
use clap_mangen::Man;
use std::path::Path;

include!("src/cli.rs");  // re-use the Cli struct definition

fn main() {
    let out_dir = std::env::var_os("OUT_DIR").unwrap();
    let cmd = Cli::command();
    let man = Man::new(cmd);
    let mut buffer = Vec::<u8>::new();
    man.render(&mut buffer).expect("man page generation failed");
    std::fs::write(Path::new(&out_dir).join("head.1"), buffer).unwrap();
}
```

After compiling, the man page is written to `$OUT_DIR/head.1`.

## Notes

- `clap_mangen` derives section headers, argument descriptions, and the NAME / SYNOPSIS sections from `clap` metadata automatically.
- `--help` output is always generated by `clap` at runtime; no build script is needed for it.
- For distribution, copy the generated `.1` file to the appropriate man page directory (e.g., `/usr/local/share/man/man1/`).
