# Terminal Readers

A Reader represents a physical device for accepting payment details in Stripe Terminal. Readers are registered to a Location and process card-present transactions.

## Signature / Usage

```bash
# Create a reader
curl https://api.stripe.com/v1/terminal/readers \
  -u "sk_test_..." \
  -d "registration_code=simulated-wpe" \
  -d "location=tml_xxx" \
  -d "label=Front Desk"
```

## Options / Props

### Create (`POST /v1/terminal/readers`)

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `registration_code` | string | Yes | Code generated by the reader during setup |
| `location` | string | Yes | ID of the Location to assign the reader to |
| `label` | string | No | Human-readable label; defaults to registration code |
| `metadata` | object | No | Arbitrary key-value pairs |

### Reader Object Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier (`tmr_...`) |
| `object` | string | Always `"terminal.reader"` |
| `action` | object \| null | Most recent action performed by the reader |
| `device_type` | string | Hardware model (e.g., `bbpos_wisepos_e`, `stripe_s700`) |
| `device_sw_version` | string | Firmware version installed on the device |
| `ip_address` | string | IP address of the reader on the local network |
| `label` | string | Human-readable label |
| `last_seen_at` | timestamp | Unix timestamp of last connectivity check |
| `livemode` | boolean | `true` in live mode, `false` in test mode |
| `location` | string | ID of the assigned Location |
| `serial_number` | string | Unique hardware serial number |
| `status` | string | `"online"` or `"offline"` |

### Action Object (`action`)

| Field | Description |
|-------|-------------|
| `type` | `collect_payment_method`, `process_payment_intent`, `confirm_payment_intent`, `refund_payment`, `set_reader_display`, `collect_inputs`, `print_content`, `process_setup_intent` |
| `status` | `"in_progress"`, `"succeeded"`, or `"failed"` |
| `failure_code` | Present when `status` is `"failed"` |
| `failure_message` | Human-readable failure description |

## Payment Action Endpoints

| Endpoint | Purpose |
|----------|---------|
| `POST /v1/terminal/readers/:id/process_payment_intent` | Hand off a PaymentIntent to the reader (server-driven) |
| `POST /v1/terminal/readers/:id/confirm_payment_intent` | Confirm a PaymentIntent on the reader (server-driven) |
| `POST /v1/terminal/readers/:id/collect_payment_method` | Collect card details via the reader (server-driven) |
| `POST /v1/terminal/readers/:id/process_setup_intent` | Hand off a SetupIntent for card saving |
| `POST /v1/terminal/readers/:id/refund_payment` | Initiate an in-person refund |
| `POST /v1/terminal/readers/:id/cancel_action` | Cancel the current in-progress action |
| `POST /v1/terminal/readers/:id/set_reader_display` | Display cart line items on a smart reader screen |

## Notes

- Use `DELETE /v1/terminal/readers/:id` to deregister a reader; this does not affect hardware
- Use `GET /v1/terminal/readers` with `location` and `status` filters to list readers at a site
- Test readers can be created with `registration_code=simulated-wpe` (WisePOS E) or `simulated-stripe-s700`
- Webhook events: `terminal.reader.action_failed`, `terminal.reader.action_succeeded`, `terminal.reader.action_updated`
