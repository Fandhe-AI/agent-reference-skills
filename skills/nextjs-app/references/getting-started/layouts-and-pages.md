# Layouts and Pages

Create pages and layouts with file-system based routing, and link between them with the `Link` component.

## Signature / Usage

```tsx
// app/blog/[slug]/page.tsx — dynamic segment + params
export default async function BlogPostPage({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const post = await getPost(slug)
  return (
    <div>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </div>
  )
}
```

```tsx
// app/ui/post.tsx — linking with next/link
import Link from 'next/link'

export default async function Posts() {
  const posts = await getPosts()
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.slug}>
          <Link href={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  )
}
```

## Options / Props

| Name | Type | Description |
|------|------|-------------|
| `params` | `Promise<{ [key]: string }>` | Dynamic segment values, must be `await`-ed |
| `searchParams` | `Promise<{ [key]: string \| string[] }>` | URL query params in a Server Component page; opts the page into dynamic rendering |
| `PageProps<'/route'>` | type helper | Global helper typing `params`/`searchParams` for `page` components (generated by `next dev`/`build`/`typegen`) |
| `LayoutProps<'/route'>` | type helper | Global helper typing `children` and named slots for `layout` components |

## Notes

- Root layout (`app/layout.tsx`) is required and must contain `<html>` and `<body>`
- Layouts nest via `children`; state is preserved across navigation, layouts do not rerender
- `useSearchParams` (Client Component hook) reads search params on the client without the dynamic-rendering opt-in that the `searchParams` prop causes
- `<Link>` extends `<a>` with prefetching and client-side navigation; prefer it over the useRouter hook for standard navigation
