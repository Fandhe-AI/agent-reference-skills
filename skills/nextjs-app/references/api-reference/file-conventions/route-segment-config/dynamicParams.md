# dynamicParams

Controls what happens when a dynamic segment is visited that was not generated by `generateStaticParams`.

## Signature / Usage

```tsx filename="layout.tsx | page.tsx"
export const dynamicParams = true // true | false
```

## Options / Props

| Value | Description |
|------|-------------|
| `true` (default) | Dynamic route segments not included in `generateStaticParams` are generated at request time. |
| `false` | Dynamic route segments not included in `generateStaticParams` return a 404. |

## Notes

- Replaces the `fallback: true | false | blocking` option of `getStaticPaths` from the `pages` directory.
- Not available when [Cache Components](../../config/next-config-js/cacheComponents.md) is enabled.
- Removed entirely (along with `dynamic`, `revalidate`, `fetchCache`) when Cache Components is enabled, as of `v16.0.0`.
