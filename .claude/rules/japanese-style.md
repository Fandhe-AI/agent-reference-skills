# 日本語表記スタイル

リポジトリ内で日本語を書くときの統一ルール。

## 基本方針

- 体言止め基調・簡潔。説明的な「〜です」「〜します」は避ける
- 句読点は「、」「。」を使用する（「，」「．」は使わない）
- 技術用語・API 名・コマンド・パッケージ名は原語（英語）を維持する
- 絵文字は使わない。ただしレポートのステータス記号（✅ ⚠️ ❌）は可
- カタカナ語末の長音は慣例に従う（「エラー」「サーバー」「フォルダー」）
- コミットメッセージは Conventional Commits 形式（詳細は create-commit スキルに従う）

## 表記揺れの統一例

| 採用 | 不採用 |
| --- | --- |
| サブエージェント | サブ・エージェント、sub agent |
| リファレンス | リファレンス（揺れなし。「参照」は日本語文脈でのみ可） |
| フロントマター | フロントマッター、front matter（本文中の指示語として）|
| インデックス | 索引（技術文脈では「索引」も可、統一を優先） |
| スキル | skill（英語表記との混在は避ける） |
| エージェント | Agent（英語固有名詞として使う場合を除く） |

## 技術用語の扱い

- ファイルパス・コマンド・コードは必ずコードブロックかインラインコードで示す
- 英語 API 名を日本語文中に交ぜる場合、前後にスペースを入れる（例: `useForm` を使って登録する）
- ドキュメント原文が英語のリファレンスページ本文は英語のまま書く（japanese-style は SKILL.md・ルール・エージェント定義に適用）
